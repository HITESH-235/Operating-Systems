Critical Section Problem
- it represents the porion of the code or a block where a process or thread accesses a shared memory
- its a section that needs to be executed in an atomic manner
- to maintain data integrity, and avoid conflicts, only one process is allowed to do so
- this ensures no two processes interfere with each other and modify shared resource simultaneuosly

Process Synchronisation
- the techniques and mechanism used to coordinate the execution of processes or threads
- it ensures the threads cooperate and communicate effectively
- prevents race condition, deadlocks, inconsistencies

Key requirements for a synchronisation mechanism:
- Mutex / Mutual Exclusion
- Progress
- Bounded waiting

Mechanisms for process synchronisation:
- Locks / Mutexes
- Semaphores
- Read-write locks

Deadlocks:
- A situatin where each of the computer processes waits for a resource that is being assinged another process
- none of the process gets executed since the resource it needs is held by some other process that is also waiting for a resource
- necessary conditions for deadlock to occur:
    a. Mutual Exclusion
    b. Hold and wait
    c. No preemption
    d. Circular wait
