Operating System
--------------------------------------------------------------------------------------------------------------------------
Kernel: The core part of the OS that manages hardware and system resources.

Process/Thread: A process is a program in execution; a thread is the smallest unit of execution within a process.

Memory Management: Techniques like Virtual Memory and Paging to handle RAM and storage.

Scheduling: Algorithms (e.g., FIFO, Round Robin) that decide which process runs on the CPU.

System Calls: The interface used by programs to request services from the kernel.

File System: The method for organizing and storing data on persistent storage.

Interrupts: Signals from hardware or software that alert the CPU to an event needing immediate attention.

Deadlock: A state where two or more processes are stuck waiting for each other to release resources.
--------------------------------------------------------------------------------------------------------------------------


Process Management:
--------------------------------------------------------------------------------------------------------------------------
Process Creation/Termination: mechanism for initiatlisation of new processes and ending of the running processses

Process Control Block: A data structure containing info about a process (e.g. state, program, counter, registers, open files)

Inter Process Communication: (IPC) Mechanisms that allow processes to communicate and synchronise their actions, such as
    shared memory, message passing, pipes, and sockets

Synchronisation: Mechanisms to ensure that concurrent processes or threads accesss shared resources in a controlled manner
    a. Mutex (Mutual Exclusion): A lock mechanismuse to protext a critical section, ensuring only one process can access it
    b. Semaphore: A signaling mechanism by multiple used to control accesss to a common resource by multiple processes
--------------------------------------------------------------------------------------------------------------------------


Memory Management:
--------------------------------------------------------------------------------------------------------------------------
Virtual Memory: A memory management technique, that provides an idealized view of the computer's memory to a process, 
    allowing it to use more memory than physically available, and simplifying memory addresses

Paging: A memory management schema that divides main memory into fixed size blocks (frames),
    and a program's logical address into equally sized blocks (pages)

Segmentation: A memory management scheme that divides a program into variable sized logical units called segments, 
    each representing a logical piece of the program (e.g. code, data, stack)

Swapping: The process of temporarily moving processes from main memory to secondary storage and then back to main memory
--------------------------------------------------------------------------------------------------------------------------


File System Management:
--------------------------------------------------------------------------------------------------------------------------
Device Driver: A software component that allows the operating system to interact with a specific hardware device

Buffering: A technique used to temporarily store the data in a buffer while it is being transferred from one device to another

Caching: A technique where frequently accessed data is stored in a faster memory (cache) to reduce access times

Spooling: (Simultaneous Peripheral Operations Online)  A technique for handling I/O devices like printers,
    where data is temporarily stored on disk before being sent to the device
--------------------------------------------------------------------------------------------------------------------------


Security And Protection:
--------------------------------------------------------------------------------------------------------------------------
Authentication: The process of verifying the idenitity of a user or process

Authorisation: The process of determining what resources a user or process is permitted to access
