--------------------------------------------------------------------------------------------------------------------------
- Software that acts as an intermediary between users/applications and hardware
- Manages system resources and provides services to applications
- Responsible for process management, memory management, file systems, device management, and security

Core Concepts

Kernel: Core component of the operating system
- Runs in privileged mode
- Manages hardware and system resources directly
--------------------------------------------------------
Process: A program in execution
- Contains its own memory space, resources, and execution state
--------------------------------------------------------
Thread: Smallest unit of CPU execution
- Exists within a process
- Multiple threads can share the same process resources
--------------------------------------------------------
Memory Management:
- Responsible for allocation, deallocation, and protection of memory
- Uses techniques such as virtual memory and paging
--------------------------------------------------------
Scheduling: Determines which process or thread gets CPU time
- Examples:
    - FCFS (First Come First Serve)
    - Round Robin
    - Priority Scheduling
--------------------------------------------------------
System Calls: Interface through which user programs request services from the kernel
File System: Organizes and manages data stored on secondary storage
Interrupts: Signals generated by hardware or software requiring CPU attention
Deadlock: A situation where two or more processes wait indefinitely for resources held by each other

Process Creation: Mechanism for creating new processes

Process Termination: Mechanism for ending processes and releasing their resources

Process Control Block (PCB): Kernel data structure used to manage a process
- Contains:
    - Process ID (PID)
    - Process State
    - Program Counter
    - CPU Registers
    - Scheduling Information
    - Memory Information
    - Open File Information

*Inter Process Communication (IPC): Mechanisms allowing processes to exchange data and synchronize activities
Types of IPC:
- Shared Memory
- Message Passing
- Pipes
- Sockets

Synchronization: Techniques used to control access to shared resources

Mutex (Mutual Exclusion): Lock allowing only one thread/process to enter a critical section at a time

Semaphore: Signaling mechanism used to control access to shared resources
- Can allow multiple processes depending on its count

- Responsible for managing RAM efficiently
- Allocates and deallocates memory to processes
- Provides memory protection and isolation

Virtual Memory:
- Provides the illusion of a large continuous memory space
- Allows processes to use more memory than physically available

Paging:
- Divides virtual memory into pages
- Divides physical memory into frames
- Pages are mapped to frames using page tables

Segmentation:
- Divides a program into logical units called segments
- Examples:
    - Code Segment
    - Data Segment
    - Stack Segment

Swapping:
- Process of moving processes between RAM and secondary storage
- Used when memory becomes limited

Device Driver:
- Software component that enables the OS to communicate with hardware devices

Buffering:
- Temporary storage of data during transfer between devices or processes

Caching:
- Storing frequently accessed data in faster memory to improve performance

Spooling:
- Technique where data is queued on storage before being sent to a device
- Commonly used for printers

--------------------------------------------------------------------------------------------------------------------------
CPU: all interrupts, action, requests, tasks eventually reaches here as an electrical signal
- constantly executes instruction, various algos existing to make its usage more efficient

RAM: the OS code itself along with parts of apps currently running, reside in here
- when CPU needs an instruction or data, it fetches it from here

Cache: Between layers of kernal and app memory, there are layers of cache memory
- both OS apps, and app instructions, are temporarily stored cache for quicker access to memory

Kernel: Special part that runs in a highly privileged mode and has direct access to all hardware
- its responsibilities include: 
    - process management (allocate RAM, protect regions)
    - memory (alloc/dealloc RAM, protect regions)
    - device (interact with hardware)
    - file system (organising data on storage)
    - security

Storage Hardware: the app's executable code is stored persistantly on a hard drive or soldie state drive
- when os loads the app, it reaches the necessary data block from the storage device to RAM
- this whole process involves complex interaction with the storage controller hardware

OS ABSTRACTIONS:

Process: Abstraction of a running program
Thread: Abstraction of a unit of execution
Virtual Memory: Abstraction that hides physical memory details
File: Abstraction for persistent data storage
Device: Abstraction that hides hardware-specific details
Socket: Abstraction used for network communication
--------------------------------------------------------

DATA STRUCTURES used by the OS:

Process Control Block (PCB): a data strucuture for each running process, containing its state, memor maps, open files, CPU registers, etc
Page Tables: used for virtual memory, these map virtual addresses to physical addresses
File Descriptors/ Inodes: Structures to keep track of open files and their attributes on disk
Device Driver: Software modules that know how to communicate with specific hardware devices
--------------------------------------------------------

JAVA RELEVANCE

JVM (Java Virtual Machine): Runtime environment for Java programs
- Provides abstractions for memory management and execution

java.lang.Process: Allows interaction with operating system processes
java.lang.Thread: Maps Java threads to underlying OS threads
java.io.File / java.nio.file.Path: Provide abstractions over operating system file systems

Garbage Collection: Automatic memory management performed by the JVM
